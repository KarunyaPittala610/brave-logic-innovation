Authentication By Encrypted Negative Password

Now a days cyber offence is most common, in that password cracking is one of the attacks.The attackers uses many ways to get the credentials such password cracking tool is used to steel the sensitive data,To overcome this problem we should use strongly hashed encrypted password.This method aims to provide an additional layer of security by leveraging a unique concept instead of storing or transmitting the actual password, an encrypted negative of the password is used.

How It Works:

1. Negative Password Concept: Instead of storing a password directly, a system calculates a "negative" or transformed version of the password. This negative password is encrypted and saved on the system.
2. Login Process: When you log in, you enter your password, which is then converted into its "negative" form by the system. The system encrypts this new version and checks if it matches the saved, encrypted negative password.
3. Security Advantage: Since the actual password isn’t stored, attackers can’t easily steal it. They would need to know both how the negative transformation and encryption are done to recreate the password.

Advantages

Enhanced Security: Since neither the actual password nor a simple hash of it is stored, this approach makes it more difficult for attackers.

Reduced Vulnerability: Even if the database storing these encrypted negative passwords is compromised, attackers would face an additional challenge since they’d need both the encryption key and the method of "negating" the password.

This approach could be beneficial for systems that need higher levels of security, like:Financial Systems,Medical Records,Government Databases.
